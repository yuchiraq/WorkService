# Сервис для учета работ строительной компании

Этот проект представляет собой веб-сервис, разработанный для управления внутренними процессами строительной компании. Он включает в себя аутентификацию пользователей, управление статьями (которые могут представлять проекты, новости или другие записи) и предоставляет базовую структуру для дальнейшего расширения.

## Основные технологии

- **Бэкенд:** Go (Golang) с использованием фреймворка Gin.
- **База данных:** PostgreSQL.
- **Контейнеризация:** Docker и Docker Compose для легкого развертывания и масштабирования.
- **Фронтенд:** HTML-шаблоны, обслуживаемые бэкендом, с простым CSS и JavaScript.

## Как запустить проект

Проект полностью контейнеризован, поэтому для его запуска вам потребуется только Docker и Docker Compose.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <URL вашего репозитория>
    cd <имя папки проекта>
    ```

2.  **Запустите сервис с помощью Docker Compose:**
    ```bash
    docker-compose up --build
    ```
    Эта команда соберет образы для бэкенда и базы данных и запустит контейнеры.

3.  **Откройте приложение в браузере:**
    Сервис будет доступен по адресу [http://localhost:8099](http://localhost:8099).
    Вы будете автоматически перенаправлены на страницу входа.

## Учетные данные для входа

Для входа в систему используйте следующие учетные данные, которые создаются при первом запуске сервиса:

- **Логин:** `admin`
- **Пароль:** `password`

## Реализованный функционал

- **Аутентификация пользователей:** Безопасный вход в систему по логину и паролю.
- **Базовая панель управления:** После входа пользователь попадает на страницу-заглушку панели управления.
- **Структура API:** Настроены маршруты для управления пользователями и статьями.
- **Статическая файловая система:** Обслуживание статических файлов (CSS, JS, изображения).
- **Перенаправление:** Автоматическое перенаправление с корневого пути на страницу входа.

## Дальнейшие шаги

Следующие шаги включают в себя разработку полнофункциональной панели управления для управления статьями, проектами и пользователями, как было описано в первоначальном плане.
